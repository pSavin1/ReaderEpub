# EPUB Reader (Readium-based)

## 1. Архитектура и решения

Приложение реализовано с использованием Clean Architecture, но с упором на скорость разработки и минимизацию кастомной логики.

Слои:

* UI (Jetpack Compose)
* Domain (use-cases)
* Data (работа с EPUB и настройками)

Ключевые решения:

* использование готовых компонентов вместо кастомной реализации
* упрощение хранения состояния

---

## 2. Работа с EPUB

Используется библиотека Readium.

### Почему Readium:

* предоставляет готовый рендеринг EPUB
* поддерживает навигацию, стили и структуру книги
* снижает объем собственного кода

### Подход:

* EPUB загружается и передается в Readium
* отображение происходит через встроенные механизмы библиотеки
* настройка внешнего вида через конфигурацию

### Преимущество:

* быстрый запуск разработки
* меньше потенциальных багов в рендеринге

---

## 3. Обработка ошибок

Реализованы:

* обработка ошибок загрузки EPUB
* ошибки парсинга

Особенности:

* ошибки обрабатываются на уровне data слоя
* UI показывает понятные состояния (loading / error)

---

## 4. Производительность

Основной упор:

* использование оптимизаций Readium “из коробки”
* отсутствие тяжелой кастомной логики

Дополнительно:

* минимальное хранение данных
* отсутствие сложного кеширования

UI:

* прогресс чтения отображается через горизонтальный progress bar
* это снижает нагрузку и упрощает восприятие

---

## 5. Production

### Мониторинг:

* интеграция возможна, но не является обязательной частью текущего решения

### Релизы:

* стандартный Android pipeline (Gradle)
* разделение debug/release

### Обработка ошибок:

* UI не падает при ошибках
* пользователь получает обратную связь

---

## 6. Риски и ограничения

* зависимость от Readium:

    * ограниченная гибкость кастомизации
    * сложность внесения изменений в рендеринг

* SharedPreferences:

    * не подходит для сложных структур данных
    * используется осознанно как легковесное решение

* меньше контроля над внутренней логикой EPUB

---

## 7. Сборка и запуск

### Требования:

* Android Studio

### Шаги:

1. Клонировать репозиторий
2. Открыть проект
3. Выполнить:

   ```
   ./gradlew installDebug
   ```
4. Запустить приложение
